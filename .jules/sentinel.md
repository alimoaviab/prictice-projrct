## 2025-05-09 - [Fix Authentication Timing Attack and NoSQL Injection Risk]
**Vulnerability:** The login endpoint at `school-app/app/api/auth/login/route.ts` suffered from two vulnerabilities. First, it lacked type checking on the `email` and `password` variables, leaving it open to NoSQL injection and unexpected type errors. Second, the response timing when verifying invalid users was measurably faster than verifying valid users, exposing the endpoint to user enumeration timing attacks.
**Learning:** These vulnerabilities existed because inputs from parsed JSON bodies were implicitly trusted without type validation, and password hash verification was short-circuited when no user document matched the given email.
**Prevention:** In API routes handling parsed inputs, explicitly enforce expected types (e.g., `typeof value === 'string'`). For authentication endpoints, ensure constant-time response behavior by implementing a dummy hash verification sequence when no user matches the database query.
