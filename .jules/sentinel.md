## 2025-02-23 - [Critical] Predictable Identifiers via math/rand in Go
**Vulnerability:** The Go backend was using `math/rand` to generate random values for invoice numbers, receipt identifiers, and certificate verification codes. `math/rand` produces pseudo-random numbers that are entirely predictable, which can lead to predictability in sensitive IDs and verification codes, allowing attackers to guess them.
**Learning:** In Go, identifiers that need a degree of cryptographic security and unpredictability (like certificate verification codes or sensitive financial receipt hashes) must be generated using `crypto/rand`, which interacts directly with the OS's entropy pool. We must also explicitly handle the `error` returned by `crypto/rand.Read` to avoid nil-pointer exceptions, utilizing fallbacks like nanosecond timestamps combined with specific system properties if entropy is exhausted.
**Prevention:** Always use `crypto/rand` for IDs and tokens that are either publicly visible or require security. Verify `math/rand` is explicitly only used for testing/seed generation or entirely isolated non-security operations. Ensure errors from `crypto/rand` are handled securely.
