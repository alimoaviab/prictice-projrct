## 2025-02-27 - Mitigate User Enumeration and NoSQL Injection in Login Endpoints
**Vulnerability:** The login endpoints (`school-app/app/api/auth/login/route.ts` and `super-admin-app/app/api/auth/login/route.ts`) were vulnerable to user enumeration via timing attacks because the password verification (`verifyPassword`) was skipped when a user was not found. They were also potentially vulnerable to NoSQL injection and type juggling due to missing type checks on the request body.
**Learning:** Checking for user existence and skipping password hashing operations makes it trivial to identify valid usernames/emails based on response time differences. NoSQL injection can happen if an object like `{"$ne": null}` is passed instead of a string to Mongoose.
**Prevention:** Compute a `DUMMY_HASH` once at module level and always run `verifyPassword(password, DUMMY_HASH)` when a user is not found to equalize the response time. Always enforce strict runtime type checking (`typeof email === "string"`) on user inputs before passing them to database queries.
