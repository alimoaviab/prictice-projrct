## 2024-05-28 - Insecure Pseudorandom Number Generation
**Vulnerability:** The application used `math/rand` to generate security-sensitive identifiers such as Invoice numbers, Receipt Numbers, Certificate IDs, and Certificate Verification Codes. `math/rand` generates predictable sequences, making these identifiers susceptible to prediction and enumeration attacks.
**Learning:** Using `math/rand` for anything requiring uniqueness and unpredictability in a security context (e.g., verification codes, external IDs) is a vulnerability. In Go, `crypto/rand` must be used for these purposes. When using `crypto/rand`, it is crucial to handle potential errors (e.g., when the system entropy pool is exhausted) by falling back to a reasonably unpredictable alternative like high-precision timestamps if necessary, to prevent nil pointer dereferences.
**Prevention:** Always use `crypto/rand` for generating identifiers, tokens, passwords, and verification codes. Ensure the error returned by `crypto/rand` methods is explicitly handled.
