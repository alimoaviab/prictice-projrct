🎯 HOW IT WORKS

  LOGIN FLOW:
  1. User logs in
  2. System loads active academic year
  3. JWT issued with school_id + academic_year_id
  4. All queries automatically scoped
  
  QUERY FLOW:
  1. Request authenticated
  2. Context extracted from JWT
  3. Query built with academicYearFilter()
  4. Database returns only scoped data
  5. Output sanitized before return
  
  YEAR SWITCH FLOW:
  1. Admin selects different year
  2. POST /api/academic-years/switch
  3. System validates year belongs to school
  4. New JWT issued with new year
  5. All queries now show new year data
  6. Old year data preserved (not deleted)
